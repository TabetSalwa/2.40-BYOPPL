Module for continuation-passing style inference methods

Rewriting the inference methods (enumeration, rejection sampling and impotance sampling) so that the functions get the continuation and keeps going by calling this continuation, rather than by returning in the usual way. This pattern for a function that receives the continuation (often called a ‘callback’) from the main computation and returns only by calling the continuation is called a coroutine. Made thanks to the Design and Implementation of Probabilistic Programming Languages course
